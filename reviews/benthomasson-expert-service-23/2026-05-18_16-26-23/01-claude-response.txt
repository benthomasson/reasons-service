## Summary

The diff is mostly solid — production hardening (FD limits, context budgets, error handling), public project access, Alembic setup, and single-pass mode are all reasonable additions. The main concerns are:

1. **Security surface area**: `data.router` is opened broadly to public access at the router level. Write endpoints are individually protected, but future GET endpoints will be public by default — this is a footgun.
2. **`chat.py` import ordering**: `logger` declaration splits the import section (line 17-18), suggesting a merge artifact.
3. **Broad `OSError` catch**: The `/ask` error handler catches all `OSError` subclasses, not just connection-related ones.
4. **No tests for `/ask` error handling or `single_ask`**: The error handling path and new mode parameter are untested.
5. **First Alembic migration without baseline**: Dual schema management path (schema.sql + Alembic) could diverge.
